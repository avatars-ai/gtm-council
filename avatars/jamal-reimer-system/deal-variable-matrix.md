# Deal Variable Matrix — Jamal Reimer Framework

## Purpose
Every mega-deal has variables you can manipulate to change the outcome. Most sellers think the only variable is price. This matrix maps ALL the variables and when to pull each lever.

## The Full Variable Matrix

### 1. Deal Scope Variables
| Variable | Description | When to Pull | Impact |
|---|---|---|---|
| **Problem scope** | Breadth of business problem addressed | Deal undersized vs. problem | Increases ACV by expanding solution footprint |
| **User scope** | Number of users/departments included | Single team → enterprise-wide | Increases seats/licenses; deepens adoption |
| **Geographic scope** | Regions/countries included | Customer has multi-geo presence | Increases deal size; creates switching cost |
| **Product scope** | Number of products/modules | Customer using only one product | Cross-sell; increases stickiness |
| **Integration scope** | Depth of technical integration | Customer wants embedded, not bolt-on | Increases switching cost; deepens dependency |

### 2. Financial Variables
| Variable | Description | When to Pull | Impact |
|---|---|---|---|
| **ACV** | Annual contract value | Always — start with problem scope | Primary revenue metric |
| **TCV** | Total contract value (multi-year) | Customer wants price certainty | Locks revenue; improves forecasting |
| **Payment terms** | Monthly, quarterly, annual, upfront | Customer cash-flow sensitive | Trade payment terms for commitment |
| **Pricing model** | Per-seat, per-usage, flat, hybrid | Current model misaligned with value | Aligns price to value; can unlock expansion |
| **Discount structure** | Volume, commitment, timing | Customer asking for lower price | Trade discount for commitment (never give free) |
| **Price locks** | Guaranteed pricing for future years | Customer worried about price increases | Secures multi-year; customer locks in rate |

### 3. Contract Variables
| Variable | Description | When to Pull | Impact |
|---|---|---|---|
| **Contract length** | 1, 2, 3+ years | Customer wants better pricing | Trade length for price; improves predictability |
| **Auto-renewal** | Automatic renewal clause | Customer procurement prefers | Reduces churn risk; creates inertia |
| **Expansion rights** | Pre-negotiated expansion pricing | Customer likely to grow | Removes friction for expansion; locks competitor out |
| **SLA guarantees** | Performance and uptime commitments | Customer risk-averse | Reduces perceived risk; justifies premium |
| **Exit clauses** | Terms for early termination | Customer insists on flexibility | Trade exit clause for higher commitment |
| **Co-termination** | Align multiple contracts to same renewal | Customer has multiple products | Simplifies; creates single renewal event |

### 4. Engagement Variables
| Variable | Description | When to Pull | Impact |
|---|---|---|---|
| **Executive engagement** | Your exec ↔ their exec relationship | Decision stuck at mid-management | Unlocks executive sponsorship; accelerates |
| **Implementation support** | Level of onboarding/deployment help | Customer worried about adoption | Reduces risk; increases early adoption |
| **Success criteria** | Defined metrics for success | Customer unsure of ROI | Creates accountability; basis for expansion |
| **QBR cadence** | Regular business review schedule | Post-sale relationship | Maintains engagement; surfaces expansion |
| **Dedicated resources** | Named CSM, TAM, or solutions consultant | High-value strategic account | Increases stickiness; premium service |
| **Training & enablement** | User training programme | Adoption risk is high | Drives adoption; reduces churn risk |

### 5. Competitive Variables
| Variable | Description | When to Pull | Impact |
|---|---|---|---|
| **Proof of concept** | Structured POC with success criteria | Customer evaluating competitors | Win on evidence; McMahon's 6 prerequisites apply |
| **Reference programme** | Customer references in same industry | Customer needs social proof | Builds confidence; addresses risk |
| **Unique capability** | Feature/data only you provide | Competitive bake-off | Changes decision criteria to your advantage |
| **Migration support** | Help switching from competitor | Customer has existing vendor | Reduces switching cost perception |
| **Bundling** | Package multiple products at discount | Competitor strong in one area | Expand the battlefield; leverage portfolio |

## Decision Framework: Which Lever to Pull

```
IF deal is undersized relative to problem
  → Pull SCOPE variables (problem, user, geographic, product)

IF deal is stuck on price
  → Pull CONTRACT variables (trade length for price, add SLAs)
  → Pull FINANCIAL variables (payment terms, price locks)
  → Do NOT discount without getting something in return

IF deal is stuck on decision
  → Pull ENGAGEMENT variables (exec engagement, success criteria)
  → Pull COMPETITIVE variables (POC, references)

IF deal is at risk of competitor displacement
  → Pull COMPETITIVE variables (unique capability, bundling, migration)
  → Pull SCOPE variables (expand to areas competitor can't match)

IF customer is risk-averse
  → Pull ENGAGEMENT variables (implementation, training, dedicated resources)
  → Pull CONTRACT variables (SLAs, exit clauses)
  → Pull COMPETITIVE variables (POC, references)
```

## Variable Trade Matrix

**Always trade — never give:**
| Customer Wants | You Trade For |
|---|---|
| Lower price | Longer contract, more users, upfront payment |
| Shorter contract | Higher ACV, no discount, auto-renewal |
| Free trial/POC | Defined success criteria, exec sponsor commitment, decision timeline |
| Better SLAs | Multi-year commitment, reference agreement |
| Flexible exit clause | Higher ACV to offset risk |
| Phased rollout | Commitment to full scope if Phase 1 succeeds |

## Karrass Concession Alignment
- Concessions must diminish over time (smaller each round)
- Never make the first major concession
- Never concede without getting something in return
- Use precise numbers (signal careful calculation, not round-number negotiation)
- Patience is the competitive advantage in concession dynamics
