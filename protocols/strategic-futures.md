# Strategic Futures — Category-Level Decision Framework

Strategic thinking framework for any revenue leader navigating category-level
decisions: market transitions, pricing model shifts, competitive positioning,
and long-term defensibility. This file contains the QUESTIONS to ask, not the
answers — answers depend on your specific company and market context.

---

## The Category Question

Before operational execution, the strategic question must be answered:

**"Are we creating a new category, or defending an old one?"**

| Category Creation | Category Defence |
|---|---|
| Addressing unrealised pain | Addressing known pain better than alternatives |
| Zero competitive set (initially) | Established competitive set |
| TAM expands as the category is defined | TAM is known and contested |
| Requires education before selling | Requires differentiation while selling |
| Higher risk, higher ceiling | Lower risk, contested ceiling |

Most companies oscillate between these two. The strategy must match which mode
you're in — category creation tactics fail in defence, and defence tactics fail
in creation.

## Moat Assessment

### Moat Scoring Exercise

For each moat, score 1-5 and note the direction (strengthening, stable, weakening):

| Moat Type | Question | Score 1-5 | Direction |
|---|---|---|---|
| **Data** | Do we own proprietary data that improves with scale? | | |
| **Network** | Does each new user make the product more valuable for others? | | |
| **Switching Cost** | How painful is it for a customer to leave? | | |
| **Brand** | Do buyers choose us by default in our category? | | |
| **Scale** | Does our cost per unit decrease meaningfully with volume? | | |
| **Distribution** | Do we have channels competitors cannot easily replicate? | | |
| **Ecosystem** | Do third parties build on our platform? | | |
| **Regulatory** | Do we have licences, certifications, or compliance that takes years to acquire? | | |

**Composite score:** Average across all 8. Below 2.5 = structurally vulnerable.
Above 3.5 = defensible position. The direction column matters more than the
absolute score — a 3.0 that's strengthening is better than a 4.0 that's weakening.

## Pricing Model Transition

When evaluating a shift from one pricing model to another (seat-based to consumption,
perpetual to subscription, fixed to usage-based), use this framework:

### Five Questions Before Transitioning

1. **Does the new model align value delivered with value captured?**
   If customers get more value from usage than from seats, consumption pricing
   captures that value. If value is per-user, seat-based is correct.

2. **What happens to revenue recognition during the transition?**
   Model the revenue dip. Subscription-to-consumption transitions typically show
   a 12-18 month revenue decline before the new model compounds past the old one.

3. **Can your sales team sell it?**
   Usage-based models require different selling motions (land small, expand through
   adoption) than seat-based (land at full contract value). Retraining is non-trivial.

4. **What does your customer's CFO prefer?**
   Some CFOs want predictable annual costs (subscription). Others want to pay for
   what they use (consumption). The customer's finance team's preference often
   matters more than your pricing team's logic.

5. **Does the model create expansion or contraction pressure?**
   Good consumption models expand naturally as usage grows. Bad ones contract as
   customers optimise usage to reduce cost. Design for organic expansion.

## Strategic Variable Assessment

For any major strategic initiative, assess these variables before committing:

| Variable | Question |
|---|---|
| **Time window** | How long do we have before the opportunity closes or the threat materialises? |
| **Reversibility** | Is this a Type 1 (irreversible) or Type 2 (reversible) decision? |
| **Resource cost** | What do we stop doing to fund this? Every initiative has an opportunity cost. |
| **Proof threshold** | What evidence would prove this is working? What evidence would prove it's not? |
| **Kill criteria** | Under what conditions do we stop? Define before starting, not after investing. |
| **Second-order effects** | And then what? What downstream consequences does this decision create? |
