# Learning Directory

This directory stores the skill's self-optimisation data. It populates over time
as you use the council — no setup required.

## Structure

| Subdirectory | Purpose |
|---|---|
| `engagements/` | Structured records of council sessions — what worked, what didn't, key decisions |
| `avatar-calibrations/` | Adjustments to avatar behaviour based on operator feedback |
| `promoted-examples/` | High-quality outputs promoted as few-shot references |
| `reviews/` | Post-session quality reviews |

## Root Files

| File | Purpose |
|---|---|
| `routing-updates.md` | Corrections to framework/avatar routing based on real usage |
| `operator-profile.md` | Learned preferences about the operator's style and priorities |
| `avatar-enrichment-log.md` | Record of avatar research and quality improvements |

## How It Works

The council captures learning from each session and uses it to improve routing,
sharpen avatar calibration, and build a library of proven outputs. The more you
use it, the better it gets.

**For new deployments:** This directory ships empty (except this README).
The skill works immediately without any learning data — baseline routing uses
the framework index and avatar files. Learning data adds precision over time.
